# Lords of the Flies

Дрон который автономно прошел лабиринт размеченный ArUco маркерами и распознал объекты по пути. 2-е место на хакатоне Сверх.

Делали за 3 дня, в команде 6 человек. Было адово но получилось.

## Задача

- Лабиринт 3x3x3 метра с ArUco маркерами на каждой ячейке (сетка 0.8х0.8м)
- Найти маршрут от стартовой позиции (Чебурашка) до финиша (Чебурашка с апельсином)
- По пути распознавать объекты (апельсины, чебурашки) - YOLO
- Все в автономе за 7 минут

## Как решали

- **ArUco Localization** - позиционирование по маркерам на полу
- **Лабиринт Граф** - построить граф свободных проходов между маркерами
- **А* навигация** - проложить путь в графе
- **YOLO** - распознавать объекты во время полета
- **Optical Flow + Lidar** - локализация между маркерами (когда маркер не видна в кадре)

## Главная идея

В каждой ячейке лабиринта размещен ArUco маркер (0.8х0.8м). Дрон летает на высоте 1-2м, видит маркер под собой, это дает точную координату. Потом летит к следующему маркеру в плане маршрута.

**Day 1 (пятница):** Получили регламент, разобрались с ArUco маркерами в OpenCV. К концу дня работал detector маркеров и basic drone controller.

**Day 2 (суббота):** Весь день на оборудовании. Измеряли координаты всех маркеров в лабиринте, строили граф. Обучали YOLO. К концу дня дрон уже летал от маркера к маркеру!

**Day 3 (воскресенье):** Финальный день. Добавили обработку потери маркера (optical flow спасает). Закончили граф лабиринта. Все интегрировали. На зачете прошли все маркеры, распознали 4 объекта. Плюс не было больших столкновений.

## ArUco Navigator

Камера смотрит вниз. Видит маркер -> получает координату -> летит туда.

```python
detector = ArUcoNavigator()
marker_id, position = detector.detect_marker(frame)  # ID маркера + (x,y,z)
```

Детекция работает в 30Hz. На высоте 1м видит маркер на расстояние ~3 метра (на лету).

Топики:
```
/camera/image_raw -> детекция маркеров
/aruco/detected_markers -> список видимых маркеров
/drone/current_position -> текущая координата (из маркеров)
```

## Граф Лабиринта

Ручное построение (на день 2 мы все координаты маркеров занесли):

```python
maze_graph = {
    1: {'position': (0, 0), 'neighbors': [2, 5]},        # можно идти на маркер 2 или 5
    2: {'position': (0.8, 0), 'neighbors': [1, 3, 6]},
    3: {'position': (1.6, 0), 'neighbors': [2, 4]},
    ...
    15: {'position': (1.6, 1.6), 'neighbors': [], 'is_finish': True}  # финиш
}
```

Граф создавался вручную потому что лабиринт динамический (конфигурация неизвестна заранее). На месте просто осмотрели весь лабиринт, обозначили маркеры.

## А* Навигация

Ищет кратчайший путь в графе маркеров:

```
Start (маркер 1) -> маркер 5 -> маркер 9 -> маркер 14 -> Finish (маркер 15)
```

Стоимость = расстояние между маркерами. Работает за O(E log V).

## YOLO детектор

Обучили на 200 фото объектов. Модель YOLOv3-tiny.

Распознаёт:
- Апельсин (класс 0)
- Чебурашка (класс 1)  
- Чебурашка с апельсином (класс 2)

Inference ~150мс, достаточно быстро для дрона.

## Optical Flow + Lidar

Когда маркер не видна в кадре (дрон между маркерами), используем:

```
Optical Flow: локальное движение в пиксели -> скорость
Lidar: расстояния до стен -> проверка столкновений + уточнение высоты
```

Это дает достаточно точности чтобы долететь до следующего маркера.

## Проблемы которые были

1. **ArUco детекция** - иногда плохо видел маркер если свет плохой. Решили - добавили подсветку снизу.

2. **Калибровка камеры** - вначале координаты были неточные. Потом переснимали калибровку, стало лучше.

3. **YOLO путал объекты** - апельсин и чебурашку иногда путал. Обучили еще на 100 фото - помогло.

4. **Optical Flow** - когда дрон слишком быстро летит, теряет трекинг. Ограничили скорость до 1 м/с (требование регламента).

5. **Граф лабиринта** - вначале неправильно измеряли расстояния. На день 2 переделали - все ок.

## Технические детали

- **Камера:** Downward-facing USB камера (1280x720, 30Hz)
- **ArUco:** OpenCV DICT_5X5_100, маркеры 0.8x0.8м
- **YOLO:** YOLOv3-tiny, 416x416, mAP ~0.78
- **Optical Flow:** PX4Flow сенсор через MAVLink
- **Lidar:** 2D Livox, 100Hz, 0.1-30м range
- **Дрон:** Сверх (250mm), Pixhawk 4 Flight Controller
